My Work


This is some of my projects I had worked on:



Intern Developper Software Robotics at Sami Agtech


Sami is an Industrial Multifunctional Agricultural System. Its aim is to reap fruit and vegetables in fields. The ultimate objective would be to achieve a return equal to or greater manual harvests, to enable American farms to cope with the lack of labour force. My assignments on this project were related to the robotics part. Programming is done with the object oriented language C#. We also have a simulation tool Roboguide (proposed by Fanuc) where SAMI is modelled. For communication between the system and the programme, we use an MQTT server. SAMI also has an AI part, linked to computer vision, which allows it to recognize the vegetables, and send the robotic arms to those positions. This project is my final internship that completes my engineering course at Polytech Montpellier.




Integration of machine learning algorithms into drones and rovers.


The objective here was to respond to a rescue scenario in an area of risk (e.g. armed conflict). To do this, we first created two models via tensorflow’s tflite model maker library. The first is able to detect a person to be rescued, and the second to detect the person via a camera view of the EP Core rover, in order to better position itself in relation to it. Then we deployed them: the first was used in a raspberry pi: the drone sends a video stream to it, and this microcontroller analyzes the images of this stream, via the object detection created. If a target has been detected, we'll send the data back to the rovers. We then used the rover’s SDK to apply the model to the video from its camera. The aim here is to adjust the position of the robot in order to allow it to precisely catch, with the help of its pliers, the figurine representing a person in danger.




Software Developpement of a model for earthquakes study bench


This project was carried out in partnership with the Montpellier Geoscience team. The aim was to create a microseism study bench. This project was divided into two parts: the creation of a code on Teensy 4.1 (C++ language) allowing the reading of data from IS3DWB sensors (high precision accelerometers) at a frequency of 10 kHz, pseudo-simultaneous, and also the creation of an IHM Python allowing better data manipulation. For the HMI, I used Tkinter, allowing to make interfaces and to use other libraries such as Matplotlib. In addition to that, I used threads to allow the application to run smoothly because we had to handle the arrival of a lot of data. The reception is done with Coolterm, because it accepts a baudrate of 2 000 000 Bits/s. The results obtained were satisfactory, the acquisition frequency was 11 kHz, which allowed the geologists to better study the precursor phenomena.


Software and Hardware Developpement of the video game Pong


In this project, we had to make a simple video game, coded in C, with a hardware in VHDL. This game had to run on an FPGA, here Basys 3. I chose to make the game PONG, which was one of the first arcade video games because it also reminded me of my childhood. It’s a simple game, but to which I brought several difficulties, a multiplayer mode and a solo mode.




Participation in the Cohoma II challenge.


Since 2021, Polytech Montpellier has been participating in the Cohoma Challenge, in partnership with the French Army. The aim is to offer an innovative, semi-automated technical solution suitable for military missions. I had the chance to participate in one of the editions of this challenge. My mission was to make a correction of GPS coordinates by merging them with the data of an inertial power station. The new data needed to be easily accessible, so we turned to ROS2. So I made several codes: two programs allowed us to extract the raw GNGGA coordinates and the IMU data by publishing them on two topics. A third one was merging data through Kalman's filter. Finally, I created a launch file, with the aim of simplifying the use of these algorithms.




Client-middleware-server system for sending remote commands to a robotic arm.


This project consisted of sending a command in position from a server to a robotic arm, simulated on CoppeliaSim. The first server sends the commands to a middleware, which will introduce a delay in the command transmission and sends the same command back to the CoppeliaSim server. Subsequently, I improved the system by allowing the user, via the ‘d’ and ‘q’ keys, to control the position of the arm. This project was carried out under the Linux environment, and allowed us to put some concepts of networking in parallel with automation. The aim was to obtain a system that would respond to commands as quickly as possible, with no signs of instability or overshooting.
